# AI Agent Rules

## 1. Code Quality

- Always write clean, modular, and maintainable code.
- Follow SOLID principles.
- Prefer reusable components and functions.
- Avoid unnecessary complexity.

## 2. Architecture Awareness

- Respect the project structure.
- Do not introduce new patterns unless necessary.
- Follow existing framework conventions.

## 3. Security First

- Never expose secrets or API keys.
- Validate all user inputs.
- Sanitize database queries.
- Apply authentication and authorization checks.

## 4. Performance

- Avoid unnecessary loops and heavy operations.
- Prefer efficient algorithms and data structures.
- Optimize network and database calls.

## 5. Error Handling

- Always implement proper error handling.
- Log meaningful errors.
- Avoid silent failures.

## 6. Documentation

- Comment complex logic.
- Provide clear README instructions when adding features.
- Explain architectural changes.

## 7. Dependency Management

- Avoid unnecessary dependencies.
- Use stable and well-maintained libraries.

## 8. Testing

- Encourage unit tests for critical logic.
- Validate edge cases.

## 9. Consistency

- Maintain consistent naming conventions.
- Follow project formatting rules.

## 10. Respect Scope

- Only modify files relevant to the task.
- Do not rewrite unrelated modules.
